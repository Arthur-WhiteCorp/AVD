Here are 50 SQL queries for the given product database schema, each with a short explanation of its purpose.

### Basic Product Queries

1. **Get all products with their basic information**
```sql
SELECT p.Pid, p.ASIN, pi.title, pi.p_group, pi.salesrank, pi.avg_rating
FROM PRODUCT p
JOIN PRODUCT_INFO pi ON p.Pid = pi.Pid;
```
*Explanation:* Retrieves the complete product catalog showing ID, ASIN, title, group, sales rank, and average rating.

2. **Find products that have no associated product info**
```sql
SELECT p.Pid, p.ASIN
FROM PRODUCT p
LEFT JOIN PRODUCT_INFO pi ON p.Pid = pi.Pid
WHERE pi.Pid IS NULL;
```
*Explanation:* Identifies products that exist in the PRODUCT table but lack detailed information in PRODUCT_INFO.

3. **Search for products with 'Harry Potter' in the title**
```sql
SELECT Pid, ASIN, title
FROM PRODUCT p
JOIN PRODUCT_INFO pi ON p.Pid = pi.Pid
WHERE pi.title LIKE '%Harry Potter%';
```
*Explanation:* Performs a case-sensitive substring search for book titles containing "Harry Potter".

4. **List all distinct product groups (categories) available**
```sql
SELECT DISTINCT p_group
FROM PRODUCT_INFO
WHERE p_group IS NOT NULL
ORDER BY p_group;
```
*Explanation:* Shows the unique types of products in the database (e.g., 'Book', 'DVD', 'Music').

5. **Count total number of products per group**
```sql
SELECT p_group, COUNT(*) AS product_count
FROM PRODUCT_INFO
GROUP BY p_group
ORDER BY product_count DESC;
```
*Explanation:* Provides a breakdown of how many items exist in each product category.

10. **Median sales rank calculation using window function**
```sql
WITH ranked AS (
  SELECT salesrank, 
         ROW_NUMBER() OVER (ORDER BY salesrank) AS rn,
         COUNT(*) OVER () AS total
  FROM PRODUCT_INFO
  WHERE salesrank > 0
)
SELECT AVG(salesrank) AS median_salesrank
FROM ranked
WHERE rn IN (FLOOR((total+1)/2), CEIL((total+1)/2));
```
*Explanation:* Computes the median sales rank across all products, which is more robust to outliers than average.

12. **Get the full path of a specific category (recursive query)**
```sql
WITH RECURSIVE cat_path AS (
  SELECT cat_id, cat_nome, pcat_id, cat_nome AS full_path
  FROM CATEGORY
  WHERE cat_nome = 'Books'  -- starting category
  UNION ALL
  SELECT c.cat_id, c.cat_nome, c.pcat_id, CONCAT(cp.full_path, ' > ', c.cat_nome)
  FROM CATEGORY c
  JOIN cat_path cp ON c.pcat_id = cp.cat_id
)
SELECT * FROM cat_path;
```
*Explanation:* Shows the entire hierarchy path from a root category down to all subcategories (using recursive CTE).

34. **Correlation between number of similar products and sales rank**
```sql
SELECT sim_count_range, AVG(pi.salesrank) AS avg_salesrank, COUNT(*) AS product_count
FROM (
  SELECT p.Pid, COUNT(sp.ASIN_SIM) AS sim_count,
         CASE
           WHEN COUNT(sp.ASIN_SIM) = 0 THEN '0 similar'
           WHEN COUNT(sp.ASIN_SIM) BETWEEN 1 AND 5 THEN '1-5 similar'
           WHEN COUNT(sp.ASIN_SIM) BETWEEN 6 AND 20 THEN '6-20 similar'
           ELSE '>20 similar'
         END AS sim_count_range
  FROM PRODUCT p
  LEFT JOIN SIMILAR_PRODUCT sp ON p.Pid = sp.Pid
  GROUP BY p.Pid
) sim_stats
JOIN PRODUCT_INFO pi ON sim_stats.Pid = pi.Pid
WHERE pi.salesrank > 0
GROUP BY sim_count_range
ORDER BY avg_salesrank;
```
*Explanation:* Investigates whether having more similar/recommended products correlates with better sales rank.
